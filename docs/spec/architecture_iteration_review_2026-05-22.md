# Architecture Iteration Review, 2026-05-22

## Current Boundary

The current evidence supports a bounded native nervous-interface mechanism for
software-agent command selection under controlled repository-debugging tasks.
It supports structured receptors, NSI latent fusion, bounded continuation state,
Debug Cortex routing, and native heads as an alternative to text-only motor
serialization.

It does not yet prove production autonomy, unrestricted shell generation,
open-ended software repair, independent external reproduction, or epoch-making
architecture status. The paper and diagrams must keep that boundary visible.

## Why The Direction Is Still Coherent

The architecture is testing a specific claim: runtime-visible system state can
be transformed into native policy state and bounded action heads before
serialization. This is distinct from prompt engineering or JSON SFT because the
model objective is a constrained native decision, not free-form text emission.

The strongest evidence layers already test source-overlap pressure, ablations,
sealed final transfer, multiple local backbones, multiple seeds, and dynamic
public-trace pressure. The remaining gap is not another figure or a larger
adapter; it is whether the same native-interface idea helps in preregistered
open-ended repair with edits, tests, rollback, and stop conditions.

## Next Architecture Iteration

Phase2S should be treated as a falsification benchmark, not as a paper-claim
upgrade by default. Later Phase2T follows the same rule: it adds useful bounded
command-selection and zero-control-caveat evidence, but it has not satisfied the
repair-loop claim gate because patch proposal, test execution, rollback,
stop-condition metrics, and a budget-matched modern coding-agent baseline are
still not complete. The next mechanism should add only the pieces required for
sandboxed repair:

- Patch proposal head for bounded edit candidates.
- Test-selection head for bounded verification commands.
- Rollback and safety head for stopping or reverting unsafe edits.
- Verification-state receptors for failing tests, changed files, and patch
  application results.
- Incident and timeout receptors so failure to make progress is a measurable
  policy output, not an implicit hang.

The benchmark must include modern agent-loop baselines and must report failure
rates, rollback precision, unsafe-command blocking, test-pass recovery, and
false-completion rate. Passing bounded command selection is not enough to claim
open-ended repair.

## Non-Negotiable Gates

- No sealed-v3 failure feedback into data, sampling, hyperparameters, or seed
  selection.
- No hardcoded test names, candidate indices, gold hints, or repository-specific
  shortcut rules.
- No broad architecture claims until preregistered repair-loop evidence,
  external public holdout, multi-seed, multi-model, budget-matched modern-agent
  baselines, safety gates, and independent reproduction all pass.
- No "epoch-making" architecture claim from reviewer agreement alone. The
  internal reviewer simulation must be unanimous, but it is only a final
  critique layer after measured repair-loop, safety, public holdout,
  multi-seed, multi-model, and independent reproduction gates pass.
- Negative evidence remains in the paper instead of being reinterpreted after
  later successes.
